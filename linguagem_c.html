A linguagem C

Aqui falaremos um pouco sobre a linguagem e a sua história, nada de mais

Existe A e B?

B existiu, porém caíram em desuso. Ela foi utilizada em universidades e foi a precessora de C. Um fato a destacar é que a linguagem B foi escrita por Ken Thompson, que, junto a Dennis Ritche, criador da linguagem C, desenvolveram o sistema operacional Unix.

Tudo é matemática?

Com um olhar mais abstrato: não, já que existem várias formas de processar dados dentro da linguagem. Porém olhando pela parte do código gerado, basicamente sim, e a esmagadora parte de todas as instruções do processador são explicadas utilizando aritimética simples. Exemplo com a arquitetura RISCV

O ilustre ";"

Um dos erros mais básicos é esquecer de colocar o ; ao final de cada linha. Mas porquê que ele existe? A resposta não é muito clara, mas podemos ver com um exemplo

imagine um mundo onde C não tem ;, então temos o código abaixo

char * str = strcopy(s)
(s + 2) = '\n'

Normal até agora, certo? Tudo que esse pedaço de código faz é copiar uma string e colocar um \n no 3o caractere. Mas o problema aparece no compilador, pois ele ignora espaços em branco, fazendo o código se parecer mais com:

char*str=strcopy(s)(s+2)='\n'

com isso, o compilador entra em ambiguidade com dois casos

  1. O programador queria chamar uma funcão que foi retornada por outra função
  2. O programador queria chamar uma função e depois calcular o valor de s + 2

como o compilador não pode "adivinhar", ele vai pela 1ª opção, que é a sintaticamente correta, mas não a desejada

por isso o ; está aqui, para tirar ambiguidade de alguns casos

Ponteiros, que nem meu dedo?

Sim! é basicamente uma variável que é capaz de buscar um valor dentro da memória, nada mais.

Eles funcionam como um nome, Já que toda vez que referimos ao nome, não estamos falando do nome em si, mas do que aquele nome remete na realidade

O que tem antes da main?

A main contém todo o programa que o programador escreve, mas não todo código que é executado, pois existe um mundo inteiro de bibliotecas sendo inicializadas por trás das cortinas que é a main, do mesmo jeito que tem muito código que é executado depois que a main retorna
